# 키맨 없이는 AI 교육이 작동하지 않는다

> 2026-04-10 | 인터뷰에서 새로 발견된 가설

---

## 발견

AI 교육이 실무에 정착하려면 **내부에 소프트웨어를 이해하는 "키맨"**이 반드시 존재해야 한다. 키맨 없이 외부 교육만으로는 실무 적용이 사실상 불가능하다.

## 증거

### 현대차 (박성민 매니저) — 가장 직접적
> "해가지고 쓰이는 프로젝트가 몇 개나 되냐면 사실상 거의 전멸. **제가 만든 거 빼고는.**"

- 박성민 본인(소프트웨어 출신, 7년 교육업계 경력)이 만든 것만 실무에 활용됨
- 비개발 직군이 교육 후 만든 것은 전멸
- 현재 자신의 팀을 "클로드 코드로 업무 자동화하는 팀으로 전환시키고 있다" — 키맨 주도의 내부 전환

### 우리투자증권 (김영종 상무)
- 교육을 위한 도구/플랫폼을 "만들어보려고 하는데 개발 능력이 높지 않아서 천천히" — 키맨 부재 시 한계
- 중견업체의 "인력 이탈"이 문제 — 키맨이 빠지면 역량이 사라짐

### 삼성증권
- 해커톤에서 비개발자는 "기획자 역할"만 담당, 실제 코딩은 외부 전문가가 대행
- 내부 AI(패브릭스) 개발은 삼성 그룹 차원의 별도 전문 조직에서 수행

## 키맨의 역할

```
키맨이 하는 것:
1. 교육 내용을 실제 업무에 "번역"
2. 보안/인프라 제약 내에서 활용 가능한 방법 찾기
3. 동료들에게 지속적 후속 지원 제공
4. 결과물의 품질 보증

키맨 없이 일어나는 것:
1. 교육 → 흥미 → 현업 복귀 → "어떻게 하더라?" → 포기
2. 외부 강사 떠남 → 물어볼 곳 없음 → 포기
3. 결과물 품질 부족 → "AI는 별로" 인식 형성 → 포기
```

## 시사점

### 제품/서비스 설계
- **우리 솔루션이 "키맨 역할"을 대체할 수 있는가?** — 핵심 검증 과제
- 플랫폼이 "교육 후 물어볼 곳" 역할을 할 수 있다면 키맨 부재를 부분적으로 보완 가능
- 또는 "키맨을 빠르게 양성하는 것"이 솔루션의 핵심 가치 제안이 될 수 있음

### 타겟팅
- 키맨이 이미 있는 조직: 바이브코딩 플랫폼 도입 + 교육 = 빠른 확산 가능
- 키맨이 없는 조직: 교육만으로는 안 되고, 컨설팅 결합 필수

### 가격/모델
- 키맨 양성 과정(고가, 장기) + 전사 확산 도구(저가, 지속) 2단계 모델 검토
