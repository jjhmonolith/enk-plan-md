# ENK 프로젝트 계획 관리

## 📋 문서화 지침

**모든 사고, 논의, 분석, 인터뷰 작업 시 반드시 준수해야 할 원칙:**

### 🔄 실시간 기록 원칙
- 모든 사고과정과 논의는 **작업 중에** 실시간으로 기록
- 분석 진행 중 중간 결과물도 즉시 문서화
- 인터뷰 진행 시 핵심 인사이트는 현장에서 바로 기록

### 📁 적절한 위치 선택
```
enk-plan-md/
├── 01-knowledge-base/     → 시장조사, 사용자조사, 경쟁사분석
├── 02-insights/          → 데이터분석, 사용자피드백, 학습내용, 추천사항
├── 03-planning/          → 제품/사업계획, 로드맵, 스펙
├── 04-meetings/          → 주간미팅, 계획세션, 이해관계자미팅
├── 05-interviews/        → 사용자/전문가/이해관계자 인터뷰
└── templates/           → 재사용 가능한 템플릿
```

### ✅ 기록해야 할 내용
1. **사고과정**: 왜 이런 결론에 도달했는가?
2. **분석방법**: 어떤 방식으로 분석했는가?
3. **중간결과**: 진행 중 발견한 인사이트
4. **변경사항**: 방향 전환이나 수정된 부분
5. **다음단계**: 향후 진행할 작업 계획

### 🎯 업데이트 빈도
- **매일**: 진행한 작업의 핵심 내용 기록
- **주간**: 주요 진전사항과 학습내용 정리
- **프로젝트 단계별**: 완료된 분석이나 인터뷰 결과 종합

### 📝 문서 작성 팁
- **구체적이고 실행가능한** 내용 위주
- **날짜와 컨텍스트** 명시
- **연결고리** 만들기 (다른 문서 참조)
- **시각적 요소** 활용 (표, 그래프, 다이어그램)

## 📊 폴더 구조 상세

### 01-knowledge-base/
- `market-research/`: 시장 동향, 규모, 트렌드 분석
- `user-research/`: 사용자 니즈, 페인포인트, 행동패턴
- `competitor-analysis/`: 경쟁사 제품, 전략, 강약점 분석
- `technology-research/`: 기술 트렌드, 솔루션 조사
- `industry-trends/`: 업계 동향, 규제, 이슈

### 02-insights/
- `data-analysis/`: 데이터 분석 결과와 해석
- `user-feedback/`: 사용자 피드백 종합 및 패턴 분석
- `lessons-learned/`: 프로젝트 진행 중 학습한 내용
- `recommendations/`: 분석 기반 추천사항과 제안

### 03-planning/
- `product/`: 제품 기획, 기능 정의, UX/UI 가이드
- `business/`: 사업 모델, 수익화 전략, 파트너십
- `growth/`: 성장 전략, 마케팅, 확산 계획
- `roadmaps/`: 개발 로드맵, 마일스톤, 일정
- `specifications/`: 기술 스펙, 아키텍처, 요구사항

### 04-meetings/
- `weekly-meetings/`: 주간 진행상황 리뷰
- `planning-sessions/`: 계획 수립 세션
- `stakeholder-meetings/`: 이해관계자 미팅
- `retrospectives/`: 회고 및 개선방안 도출

### 05-interviews/
- `user-interviews/`: 실제 사용자 인터뷰
- `expert-interviews/`: 도메인 전문가 인터뷰  
- `stakeholder-interviews/`: 주요 이해관계자 인터뷰
