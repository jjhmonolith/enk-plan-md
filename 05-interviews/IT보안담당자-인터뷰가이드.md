# IT/보안 담당자 인터뷰 가이드

## 인터뷰 대상
- **기업 규모**: 500명 이상 (보안 정책이 체계화된 기업)
- **업종**: 금융, 제조업 등 보안이 중요한 업종
- **직책**: IT 담당자, 정보보안 담당자, 시스템 관리자
- **경험**: 망분리 환경 운영, AI 도구 도입 검토 경험

## 핵심 검증 가설
- **H3-1**: 보안이 까다로운 환경에서도 바이브코딩을 활용할 방법이 있다
- **기술적 실현 가능성**: 망분리, 데이터 보안, 접근 통제 하에서의 AI 교육 가능성

---

## 1단계: 스크리닝 & 기술 환경 파악 (15분)

### 기본 정보 확인
1. **회사 규모와 업종, 그리고 담당하시는 업무 영역을 알려주세요.**

2. **현재 회사의 보안 정책 수준은 어느 정도인가요?**
   - 망분리 환경인가요?
   - 외부 인터넷 접속이 제한되나요?
   - 개인 기기 사용이 가능한가요?

3. **직원들이 업무에서 사용할 수 있는 AI 도구가 있나요?**
   - ChatGPT, Microsoft Copilot 등
   - 사용 제한이나 정책이 있다면?

### 현재 AI 도구 현황

4. **회사에서 AI 도구 도입을 검토해보신 적이 있나요?**
   - 도입했다면: 어떤 도구를, 어떤 과정을 거쳐서?
   - 검토만 했다면: 도입하지 않은 이유는?

5. **AI 도구 도입 시 가장 큰 우려사항이 무엇인가요?**
   
   **🔍 예상 우려사항**:
   - 데이터 유출, 개인정보 보호, 정보보안, 업무 의존도, 정확성

---

## 2단계: 보안 제약사항 구체화 (25분)

### 망분리 환경에서의 제약

6. **망분리 환경에서 직원들이 외부 AI 서비스를 사용하려면 어떤 절차를 거쳐야 하나요?**
   - 불가능한가요?
   - 가능하다면 어떤 조건 하에서?

7. **내부망에서만 사용 가능한 AI 솔루션을 도입한다면 어떤 조건들이 필요한가요?**
   - 온프레미스 설치?
   - 데이터 처리 방식?
   - 로그 관리?

   **🔍 기술적 실현 가능성**: 우리 솔루션의 온프레미스 버전 필요성

### 데이터 보안 요구사항

8. **직원들이 AI 도구에 입력하는 데이터에 대한 제약이 있나요?**
   - 업무 데이터 입력 금지?
   - 개인정보 포함 데이터 제한?
   - 기밀 정보 분류 기준?

9. **AI 도구로 생성한 결과물(코드, 문서 등)에 대한 보안 검토나 승인 과정이 필요한가요?**

10. **만약 교육 목적이라면 데이터 보안 요구사항이 달라질까요?**
    
    **🔍 교육 예외**: 실제 업무 데이터가 아닌 샘플 데이터 사용 시 완화 가능성

### 접근 통제 및 모니터링

11. **AI 도구 사용에 대한 모니터링이나 로깅이 필요한가요?**
    - 사용자 추적? 사용 내용 기록? 결과물 저장?

12. **특정 직급이나 부서만 AI 도구를 사용할 수 있도록 제한해야 하나요?**

---

## 3단계: 바이브코딩 특화 검증 (20분)

### 바이브코딩 도구의 특성 이해

**바이브코딩 설명**: 
"바이브코딩은 자연어로 대화하면서 웹앱이나 업무 도구를 만들 수 있는 AI 개발 플랫폼입니다. 실제 코딩 지식 없이도 챗봇, 계산기, 데이터 분석 도구 등을 만들 수 있습니다."

13. **이런 도구에 대한 첫 반응은 어떠신가요? 보안 관점에서 우려되는 부분이 있나요?**

14. **직원들이 만든 웹앱이나 도구를 회사에서 실제로 사용하려면 어떤 검토나 승인 과정을 거쳐야 하나요?**
    
    **🔍 실무 적용 가능성**: 만든 것을 실제로 쓸 수 있는가?

15. **바이브코딩으로 만든 결과물이 외부 서버에 호스팅되는 것에 대해 어떻게 생각하시나요?**
    - 완전 금지? 조건부 허용? 내부 호스팅 필요?

### 해결 방안 탐색

16. **만약 이런 도구를 교육 목적으로만 사용한다면 (실제 업무 적용 X), 보안 요구사항이 달라질까요?**

17. **온프레미스나 프라이빗 클라우드 환경에서 이런 서비스를 제공할 수 있다면 어떨까요?**
    
    **🔍 기술적 대안**: 우리가 제공해야 할 기술적 솔루션

18. **교육용 샘플 데이터만 사용하고, 만든 결과물은 교육 종료 후 삭제한다면 허용 가능할까요?**

---

## 4단계: 해결책 및 대안 모색 (15분)

### 보안 정책 내에서의 가능성

19. **현재 보안 정책 하에서 이런 AI 교육이 가능하려면 어떤 조건들이 갖춰져야 할까요?**
    - 기술적 조건
    - 정책적 조건
    - 절차적 조건

20. **다른 회사들은 이런 문제를 어떻게 해결하고 있는지 아시나요?**
    
    **🔍 벤치마킹**: 다른 기업의 해결책 사례

21. **보안팀 승인을 받으려면 어떤 자료나 보장이 필요할까요?**
    - 보안 인증? 데이터 처리 방침? 기술적 명세?

### 현실적 타협점

22. **완벽한 솔루션이 어렵다면, 어느 정도까지 타협이 가능할까요?**
    - 제한된 기능이라도 교육은 가능?
    - 특정 조건 하에서 시범 운영?

23. **만약 외부 교육기관에서 진행하는 오프라인 교육이라면 어떨까요?**
    
    **🔍 대안 솔루션**: 사내가 아닌 외부에서의 교육 가능성

---

## 5단계: 우리 솔루션의 기술적 요구사항 (10분)

### 필수 기술 요구사항

24. **우리가 이런 서비스를 제공한다면, 보안 관점에서 반드시 갖춰야 할 기능이나 정책이 있을까요?**
    - 데이터 암호화? 접근 로그? 사용자 인증?

25. **서비스 도입 전 보안 검토 과정은 보통 어떻게 진행되나요?**
    - 필요한 문서들
    - 검토 기간
    - 승인 절차

26. **혹시 비슷한 서비스를 검토하거나 도입해본 경험이 있으시다면 공유해주실 수 있나요?**

## 6단계: 마무리 및 추가 연결 (5분)

27. **다른 회사의 IT/보안 담당자 중에서 이런 고민을 해보셨을 분이 계시다면 소개받을 수 있을까요?**

28. **마지막으로 AI 도구 도입을 고려하는 서비스 업체에게 꼭 당부하고 싶은 보안 요구사항이 있다면?**

- [ ] 데이터 보안 요구사항과 허용 범위
- [ ] 접근 통제 및 모니터링 필요사항
- [ ] 결과물 사용에 대한 승인 프로세스

### H3-1 가설 검증 결과
- [ ] **완전 불가능** vs **조건부 가능** vs **대안 필요** 중 어디에 해당하는지
- [ ] 조건부 가능하다면 필요한 구체적 조건들
- [ ] 대안이 필요하다면 어떤 대안들이 가능한지

### 기술적 솔루션 요구사항
- [ ] 온프레미스 버전 필요성
- [ ] 보안 인증 요구사항
- [ ] 데이터 처리 방침 요구사항
- [ ] 접근 통제 기능 요구사항

### 시장 진입 전략 시사점
- [ ] 보안 검토 프로세스와 소요 기간

- [ ] 우리가 차별화할 수 있는 보안 기능들

**⚠️ 주의사항**:
- **보안 담당자는 기본적으로 보수적 성향임을 인정하고 접근**
- **"왜 안 되는가"보다 "어떻게 하면 가능한가"에 초점**
- **완전한 솔루션이 아니어도 교육 목적의 제한적 사용 가능성 탐색**
- **구체적 기술 명세보다는 정책적/절차적 요구사항에 집중**
