# 비개발자 AI 해커톤 플랫폼 - 제품 비전 문서

## 제품 한 줄 요약
**기업과 교육기관이 비개발자 대상 AI 해커톤을 별도 준비 없이 바로 열 수 있는 올인원 SaaS**

## 문제 정의

### 기업의 AI 교육 딜레마
- **현황**: 모든 기업이 AI 전환(AX) 압력 하에 있음
- **기존 교육의 한계**: 수동적 학습, 높은 비용, 낮은 실무 적용률
- **경영진 요구**: "교육 수료 인원" → "AI 활용 결과물 건수"로 성과 지표 변화

### 해커톤의 잠재력과 진입장벽
- **장점**: 즉각적 교육 효과, 실질적 결과물, 상대적 저비용
- **기존 한계**: 개발자 중심 행사, 높은 기술적 진입장벽

### 바이브코딩 시대의 기회
- **변화**: ChatGPT, Claude 등 AI로 누구나 코딩 가능
- **도구**: Lovable, Bolt, Google AI Studio 등 노코드 개발 환경
- **결과**: 비개발자 AI 해커톤 가능성 열림

### 기존 도구의 한계
1. **비용 관리 불가**: API 사용량 예측 어려움
2. **교육 단계 부재**: 도구 사용법 학습 과정 없음
3. **공유/피드백 제한**: 결과물 전시 및 평가 기능 부족
4. **행사 운영 기능 부재**: 참가자 관리, 진행 통제 불가

## 솔루션 개요

### 핵심 가치 제안
**교육에서 해커톤으로의 자연스러운 전환을 통한 실질적 AI 역량 개발**

### 3가지 핵심 기능
1. **교육-실전 통합**: 가이드형 튜토리얼 → 자유 구현 (동일 에디터)
2. **운영자 관제**: 실시간 진행상황 모니터링, 원클릭 통제
3. **결과물 중심 평가**: 자동 갤러리, 체험형 피드백, 구조화된 심사

## 타겟 고객

### 1차 고객: 기업 HRD팀
- **담당자**: 대기업 인재개발팀 과장~차장급
- **페인포인트**: AI 교육 성과 보고, 예산 대비 효과 입증
- **구매동기**: 높은 참여율, 실질적 결과물, 예산 확정성
- **특히 중요한 업종**: 금융·제조업 (보안 규제로 인한 AI 도구 사용 제약)

### 초기 시장: 대학
- **시즌**: 7-8월 여름방학 해커톤
- **장점**: 기존 네트워크 활용 가능, 레퍼런스 구축
- **참가자**: 중고등학생 ~ 대학생

## 제품 아키텍처

### 해커톤 진행 플로우
```
운영자: 해커톤 생성 → 참가자 모집 → 교육 모드 시작 → 해커톤 모드 전환 → 평가 진행 → 성과 보고
참가자: 교육 수강 → 도구 익히기 → 아이디어 구현 → 결과물 제출 → 상호 평가
```

### 핵심 모듈
1. **교육 모드**: 스텝별 가이드 + AI 튜터 + 웹 기반 에디터
2. **해커톤 모드**: 자유 구현 환경 + API 사용량 관리 + 실시간 모니터링
3. **평가 모드**: 결과물 갤러리 + 체험형 피드백 + 구조화된 심사
4. **관리자 대시보드**: 진행상황 추적 + 비용 관리 + 성과 리포트

## 경쟁 환경 분석

### 기존 카테고리와의 차이점

| 카테고리 | 대표 제품 | 장점 | 한계 | 우리의 차별점 |
|---------|---------|------|------|--------------|
| 바이브코딩 도구 | Lovable, Bolt | 강력한 개발환경 | 행사운영 기능 전무 | 교육+운영+평가 통합 |
| 해커톤 플랫폼 | Devpost, HackerEarth | 완성된 행사운영 | 개발자 중심, 개발환경 부재 | 비개발자 특화, 에디터 내장 |
| 기업 교육 LMS | Cornerstone, Docebo | 체계적 교육관리 | 수동적 학습, 결과물 없음 | 능동적 창작, 실질적 산출물 |

### 우리의 포지셔닝
**바이브코딩 개발환경 + 해커톤 운영기능 + 교육 LMS 학습단계**

## 비즈니스 모델

### 수익 구조
1. **해커톤 단위 과금**: 1회 행사당 정액 요금 (플랫폼 사용료 + AI API 크레딧 포함)
2. **행사 운영 대행**: 기획, 장소, 장비, 강사, 평가위원 등 커스텀 패키지

### 가격 정책
- **투명한 비용**: 사전 총비용 확정으로 예산 품의 가능
- **포함 항목**: AI API 사용료를 패키지에 포함하여 예측 불가능한 비용 제거
- **확장성**: 참가자 수에 따른 유연한 가격 구조

## 시장 기회

### 직접 시장
- **해커톤 관리 소프트웨어**: 수억 달러 규모 (초기 단계)
- **주요 플레이어**: HackerEarth, Brightidea 등

### 인접 시장
- **기업 LMS 시장**: 145억 달러 → 723억 달러 (2034년 전망)
- **노코드/로우코드**: 212억 달러 → 1,870억 달러 (2030년 전망)

### ROI 스토리
**"연간 AI 교육 예산 3억 원 중 5천만 원을 해커톤 3회에 투입하면, 이러닝보다 높은 참여율 + 실제 결과물 수십 개"**

## 제품 로드맵

### Phase 1: MLP (Minimum Lovable Product)
**목표**: 해커톤 1회를 처음부터 끝까지 운영 가능
- 해커톤 생성/관리
- 가이드형 교육 모드 (AI 튜터 포함)
- 웹 기반 AI 에디터
- 진행 통제 (시작/종료/타이머)
- 결과물 제출 및 기본 평가
- 관리자 대시보드
- API 사용량 상한 관리

### Phase 2: UX 강화
- 카훗 스타일 대기실
- 실시간 투표/점수 집계
- 갤러리 워크 (결과물 체험)
- 개선된 피드백 시스템

### Phase 3: Enterprise 기능
- 경영진 리포트 자동 생성 (PDF/PPT)
- 팀 구성 기능
- 프론트엔드 편집 모드
- 상세 API 사용량 대시보드
- SSO 연동

### Phase 4: 글로벌 확장
- 다국어 지원
- 멀티 테넌트 (사업부별 독립 운영)
- HR 시스템 연동 (Workday, SAP SuccessFactors)
- 조직 AI 역량 지수 추적

## 핵심 가설 및 검증 계획

### 우선순위별 가설
1. **기업 HRD팀은 교육 과정보다 산출물과 평가를 더 중시한다**
   - 검증방법: HRD 담당자 인터뷰, 기존 교육 프로그램 성과 지표 분석
   
2. **비개발자 대상 AI 해커톤에 대한 기업 수요가 존재한다**
   - 검증방법: MVP를 통한 파일럿 프로그램 운영, 고객 피드백 수집
   
3. **HRD팀은 참가자보다 운영자 관점에서 제품을 평가한다**
   - 검증방법: 제품 데모 시 운영 편의성 vs 사용자 경험에 대한 반응 분석
   
4. **바이브코딩만으로는 비개발자가 유의미한 결과물을 만들기 어렵다**
   - 검증방법: 교육 단계 유무에 따른 결과물 품질 A/B 테스트

## 성공 지표 (KPI)

### 제품 지표
- 해커톤 완주율 (참가 등록 → 결과물 제출)
- 결과물 품질 점수 (표준화된 평가 기준)
- 플랫폼 사용 시간 (교육 모드 vs 해커톤 모드)
- API 사용 효율성 (비용 대비 결과물 품질)

### 비즈니스 지표
- 월간 신규 해커톤 개설 수
- 고객 재구매율 (동일 기업의 추가 해커톤)
- 평균 해커톤 규모 (참가자 수)
- 고객 생애 가치 (LTV)

### 사용자 만족도
- 운영자 만족도 (NPS)
- 참가자 만족도
- 추천 의향도
- 성과 보고 만족도 (HRD 관점)
