# MVP: 운영자 대상 영업 데모 버전

> 다음 주 우리투자증권 미팅에서 보여줄 수 있는 최소 동작 버전.
> **목적**: "이 플랫폼을 6주 교육에 쓰면 참가자들이 만든 결과물이 이렇게 공유됩니다"를 체감시키는 것.

## 데모 시나리오

미팅에서 직접 시연하는 흐름. 운영자(김영종 상무)가 옆에서 보고 있다고 가정.

### 장면 1: "참가자들이 만들고 있는 것을 한눈에"

> 갤러리 목록을 보여주며

"참가자 22명이 각자 바이브코딩으로 앱을 만들고 있으면, 이 화면에서 실시간으로 볼 수 있습니다."

- 갤러리에 3-5개의 **데모 프로젝트**가 카드로 나열됨
- 각 카드에 라이브 앱 미리보기가 돌아가고 있음
- "지금 보시는 것들이 전부 실제로 동작하는 앱입니다"

### 장면 2: "클릭하면 바로 써볼 수 있습니다"

> 카드 하나를 클릭하여 상세 페이지로 이동

"참가자가 만든 앱을 바로 체험할 수 있습니다. 별도 설치나 배포 과정 없이요."

- iframe에서 라이브 앱이 돌아감
- 실제로 클릭하고 사용해봄
- AI가 생성한 프로젝트 설명(목적, 사용법)이 보임
- "이 설명은 AI가 코드를 분석해서 자동으로 만든 겁니다"

### 장면 3: "서로 피드백을 주고받습니다"

> 리액션 버튼을 눌러봄

"참가자들이 서로 리액션을 남기면서 동기부여를 받습니다. 인기 있는 프로젝트가 자연스럽게 드러나고요."

- 🔥 👏 💡 ❤️ 리액션을 직접 눌러봄
- 카운트가 올라가는 걸 보여줌

### 장면 4: "참가자 입장에서는 이렇습니다"

> 코딩 화면(HackathonProgress)으로 전환

"참가자는 코딩하다가 준비되면 공개 토글만 켜면 됩니다."

- 공개/비공개 토글을 OFF→ON으로 전환
- AI가 설명을 자동 생성하는 과정이 보임
- "설명이 마음에 안 들면 직접 수정할 수 있고, 코드를 고친 다음에 업데이트 버튼을 누르면 됩니다"
- 갤러리로 다시 돌아가면 방금 공개한 프로젝트가 나타나 있음

### 마무리

"6주 교육이 끝나면 이 갤러리 자체가 성과 보고서가 됩니다. 22명이 만든 결과물이 다 여기에 있으니까요."

## MVP 기능 범위

데모에 필요한 것만. **빠지는 것은 과감하게 뺀다.**

### 들어가는 것

| 기능 | 설명 | 이유 |
|------|------|------|
| **갤러리 목록** | 공개 프로젝트 카드 그리드 + 라이브 iframe 미리보기 | 데모 첫 인상 |
| **갤러리 상세** | 라이브 앱 iframe (상호작용 가능) + 설명 + 리액션 | 핵심 가치 체감 |
| **공개/비공개 토글** | ON 시 갤러리 노출, OFF 시 제거 | 참가자 플로우 시연 |
| **AI 설명 자동 생성** | 공개 전환 시 코드 분석하여 목적/사용법 생성 | "AI 네이티브" 임팩트 |
| **설명 수정** | 사용자가 제목/목적/사용법 직접 편집 | 자동 생성이 완벽하지 않을 때 |
| **리액션** | 🔥 👏 💡 ❤️ 4종 토글 | 피드백 체감 |
| **데모 데이터** | 미리 만들어둔 3-5개 프로젝트 (공개 상태) | 빈 갤러리는 설득력 없음 |

### 빠지는 것 (2차 이후)

| 기능 | 빠지는 이유 |
|------|-----------|
| 댓글 | 리액션만으로 데모 충분 |
| 알림 | 실시간 시연 어려움 |
| 댓글 → AI 전달 | 댓글이 없으니 불필요 |
| 실시간 ActionCable | 새로고침으로 대체 가능 |
| 정렬/필터 | 데모 데이터 5개면 불필요 |
| [설명 업데이트] 버튼 | 데모에서 코드 수정 후 재생성까지 보여줄 필요 없음 |
| 썸네일 이미지 업로드 | iframe 미리보기가 있으므로 불필요 |

## MVP 데모 데이터

미팅 전에 미리 만들어둘 데모 프로젝트들. 실제로 포트 3000에서 돌아가는 앱이어야 함.

| # | 프로젝트명 | 설명 | 용도 |
|---|-----------|------|------|
| 1 | 증권 뉴스 수집기 | 종목 키워드로 뉴스를 모아주는 대시보드 | 금융업 맥락에 맞는 메인 데모 |
| 2 | 회의록 요약 도우미 | 텍스트를 넣으면 AI가 요약해주는 도구 | 비개발자도 만들 수 있다는 걸 보여줌 |
| 3 | 팀 일정 대시보드 | 간단한 캘린더/할일 관리 | 업무 도구 예시 |
| 4 | 환율 계산기 | 실시간 환율 조회 + 변환 | 간단하지만 실용적인 예시 |
| 5 | 데이터 시각화 도구 | CSV 업로드하면 차트 생성 | 분석 업무 예시 |

이 앱들은 바이브코딩으로 실제 만들어서 데모 계정 컨테이너에 올려둠.

# 전체 청사진 (Full Version)

> 아래는 MVP 이후의 전체 기능 청사진입니다.

---

## 배경

- 각 참가자의 앱은 `{username}.serve-domain`에 **상시 배포**되어 있음 (포트 3000, CHP 프록시)
- 별도 제출/배포 과정 없이 코딩하면 바로 반영됨
- 현재는 서로의 앱을 볼 수 있는 경로가 없음 → **갤러리가 그 경로**

---

## 유저 스토리

### 참가자 A: "내가 만든 걸 보여주고 싶다"

1. 해커톤에 참가해서 바이브코딩으로 앱을 만들고 있음
2. 어느 정도 만들어져서 다른 사람들에게 보여주고 싶음
3. 내 프로젝트 설정에서 **[공개]로 전환**
4. AI가 내 코드를 분석해서 **"이 앱은 뭐고, 이렇게 쓰면 됩니다"** 설명을 자동으로 써줌
5. 설명을 읽어보고 마음에 안 드는 부분은 직접 수정
6. 갤러리에 내 앱이 나타남
7. 다른 사람들이 리액션을 남기기 시작 → 뿌듯함, 동기부여
8. 코드를 더 개선한 뒤 **[설명 업데이트]** 버튼을 눌러 설명도 최신화

### 참가자 B: "다른 사람은 뭘 만들고 있지?"

1. 갤러리 탭을 눌러봄
2. 공개된 프로젝트들이 **카드 형태로 나열**됨 — 각 카드에 **라이브 미리보기**가 보임
3. "오 이거 재밌다" 하는 프로젝트 카드를 클릭
4. 상세 페이지에서 **실제 앱을 직접 써볼 수 있음** (iframe)
5. 설명을 읽고 어떤 목적으로 만들었는지 파악
6. 🔥 리액션을 남김
7. (2차) "여기에 이 기능 추가하면 진짜 좋겠다" 댓글을 남김

### 참가자 A: "피드백을 받아서 바로 개선하고 싶다" (2차)

1. 알림이 옴: "참가자 B가 댓글을 남겼습니다"
2. 댓글 내용: "환율 변환 기능도 추가하면 좋겠어요"
3. **[AI에게 전달]** 버튼을 누름
4. 에디터의 AI 채팅에 자동으로 프롬프트가 들어감: "사용자 피드백: 환율 변환 기능 추가 요청. 이를 반영해주세요."
5. AI가 코드를 수정 → 앱에 즉시 반영 (상시 배포)
6. 갤러리에서 보는 사람도 바로 최신 버전을 체험

### 운영자: "참가자들이 뭘 만들고 있는지 한눈에 보고 싶다"

1. 갤러리 탭에서 공개된 프로젝트 전체를 조회
2. 리액션이 많은 인기 프로젝트를 파악
3. 해커톤 진행 중에도 성과를 실시간으로 확인 가능
4. 경영진 보고 시 갤러리 화면을 직접 보여줄 수 있음

- 해커톤 상세 화면의 **탭으로 상시 노출** (진행 중이든 끝났든)

- **하단**: 제목 + 만든이 이름 + 리액션 요약
- 카드 전체가 클릭 영역 → 상세 페이지로 이동

**정렬:** 인기순(리액션 합계) / 최신순(공개 시점)

**모바일:** 2열 그리드

**공개 토글 UX:**
- 기본값: OFF (비공개)
- OFF→ON 전환 시:

  2. 제목/목적/사용법이 자동으로 채워짐

**직접 수정:**
- 제목, 목적, 사용법 모두 사용자가 자유롭게 편집 가능
- AI 생성은 초안일 뿐, 최종 내용은 사용자가 결정

## 개발 참고사항

### 데이터 모델
- **Project**: organization_id, user_id, opencode_username, is_public, title, description, usage, thumbnail_url
- **Reaction**: project_id, user_id, kind(enum: like/fire/clap/idea), unique index [project_id, user_id, kind]
- **Comment** (2차): project_id, user_id, body, is_read

### 앱 미리보기 URL
- `https://{opencode_username}.{SERVE_DOMAIN}/`
- opencode_username은 users_opencode_ids 테이블에서 매핑
- iframe에 로드, CORS 이미 허용됨 (CHP에서 `Access-Control-Allow-Origin: *`)

### AI 설명 생성
- enk-opencode API로 프로젝트 파일 조회 → LLM에 전달 → 목적/사용법 생성
- 또는 enk-opencode 내 세션 API 활용

### 기존 패턴 참고
- API: JSONAPI + authorize_user_token! (organizations_controller 참고)
- 실시간: OrganizationChannel 패턴 참고
- UI: CDS 컴포넌트 + Emotion + CreateDialog/HackathonProgress 패턴 참고
