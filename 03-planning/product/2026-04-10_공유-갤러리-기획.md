# 공유 갤러리 기획

> 2026-04-10 | ENK 해커톤 플랫폼 — 공유 갤러리 기능

---

## 한 줄 요약

**참가자들이 만들고 있는 앱을 서로 실시간으로 구경하고, 리액션과 피드백을 주고받는 공간.**

---

## 배경

- 각 참가자의 앱은 `{username}.serve-domain`에 **상시 배포**되어 있음 (포트 3000, CHP 프록시)
- 별도 제출/배포 과정 없이 코딩하면 바로 반영됨
- 현재는 서로의 앱을 볼 수 있는 경로가 없음 → **갤러리가 그 경로**

---

## 유저 스토리

### 참가자 A: "내가 만든 걸 보여주고 싶다"

1. 해커톤에 참가해서 바이브코딩으로 앱을 만들고 있음
2. 어느 정도 만들어져서 다른 사람들에게 보여주고 싶음
3. 내 프로젝트 설정에서 **[공개]로 전환**
4. AI가 내 코드를 분석해서 **"이 앱은 뭐고, 이렇게 쓰면 됩니다"** 설명을 자동으로 써줌
5. 설명을 읽어보고 마음에 안 드는 부분은 직접 수정
6. 갤러리에 내 앱이 나타남
7. 다른 사람들이 리액션을 남기기 시작 → 뿌듯함, 동기부여
8. 코드를 더 개선한 뒤 **[설명 업데이트]** 버튼을 눌러 설명도 최신화

### 참가자 B: "다른 사람은 뭘 만들고 있지?"

1. 갤러리 탭을 눌러봄
2. 공개된 프로젝트들이 **카드 형태로 나열**됨 — 각 카드에 **라이브 미리보기**가 보임
3. "오 이거 재밌다" 하는 프로젝트 카드를 클릭
4. 상세 페이지에서 **실제 앱을 직접 써볼 수 있음** (iframe)
5. 설명을 읽고 어떤 목적으로 만들었는지 파악
6. 🔥 리액션을 남김
7. (2차) "여기에 이 기능 추가하면 진짜 좋겠다" 댓글을 남김

### 참가자 A: "피드백을 받아서 바로 개선하고 싶다" (2차)

1. 알림이 옴: "참가자 B가 댓글을 남겼습니다"
2. 댓글 내용: "환율 변환 기능도 추가하면 좋겠어요"
3. **[AI에게 전달]** 버튼을 누름
4. 에디터의 AI 채팅에 자동으로 프롬프트가 들어감: "사용자 피드백: 환율 변환 기능 추가 요청. 이를 반영해주세요."
5. AI가 코드를 수정 → 앱에 즉시 반영 (상시 배포)
6. 갤러리에서 보는 사람도 바로 최신 버전을 체험

### 운영자: "참가자들이 뭘 만들고 있는지 한눈에 보고 싶다"

1. 갤러리 탭에서 공개된 프로젝트 전체를 조회
2. 리액션이 많은 인기 프로젝트를 파악
3. 해커톤 진행 중에도 성과를 실시간으로 확인 가능
4. 경영진 보고 시 갤러리 화면을 직접 보여줄 수 있음

## UX 상세

### 1. 갤러리 진입

- 해커톤 상세 화면의 **탭으로 상시 노출** (진행 중이든 끝났든)
- 탭: `진행` | `갤러리` | `참가자`
- 공개된 프로젝트가 없으면: "아직 공개된 프로젝트가 없습니다. 첫 번째로 공개해보세요!"

- 카드 전체가 클릭 영역 → 상세 페이지로 이동

**정렬:** 인기순(리액션 합계) / 최신순(공개 시점)

**모바일:** 2열 그리드

**라이브 앱 영역:**
- iframe으로 `{username}.serve-domain` 로드

**설명 영역:**
- AI가 생성한 목적 + 사용법

**리액션:**

- 같은 종류는 토글 (누르면 추가, 다시 누르면 취소)
- 내가 누른 리액션은 활성 상태로 표시

**댓글 (2차):**
- 시간순 정렬
- 본인 댓글만 삭제 가능

```
│  📋 목적                          │
│  ┌──────────────────────────────┐ │
│  │ 뉴스와 리서치 데이터를        │ │
│  │ 자동으로 수집하여...          │ │
│  └──────────────────────────────┘ │
│                                    │
│  🔧 사용법                        │
│  ┌──────────────────────────────┐ │
│  │ 1. 종목코드를 입력합니다      │ │

│  └──────────────────────────────┘ │

│                                    │
└────────────────────────────────────┘
```

**공개 토글 UX:**
- 기본값: OFF (비공개)
- OFF→ON 전환 시:
  1. "AI가 프로젝트 설명을 생성합니다" 로딩
  2. 제목/목적/사용법이 자동으로 채워짐
  3. 사용자가 확인/수정 후 [저장] → 갤러리에 노출
- ON→OFF 전환 시: 즉시 갤러리에서 제거. 설명은 그대로 보존.

**[설명 업데이트] 버튼:**
- 코드를 수정한 뒤 설명을 최신화하고 싶을 때 클릭
- AI가 현재 코드를 다시 분석하여 설명 재생성
- 기존 설명을 사용자가 수정한 상태면: "AI가 새로 생성하면 현재 내용이 대체됩니다. 계속하시겠습니까?" 확인

**직접 수정:**
- 제목, 목적, 사용법 모두 사용자가 자유롭게 편집 가능
- AI 생성은 초안일 뿐, 최종 내용은 사용자가 결정

## 구현 우선순위

### 1차: MVP

1. Project 모델 (join 시 자동 생성, is_public, title, description, usage)
2. 공개/비공개 토글
3. AI 설명 자동 생성 (비공개→공개 시)
4. 설명 직접 수정 + [설명 업데이트] 버튼
5. 갤러리 목록 (라이브 iframe 미리보기 카드)
6. 갤러리 상세 (라이브 앱 체험 + 설명)
7. 리액션 4종 (토글)

### 2차

8. 댓글
9. 알림 (내 프로젝트에 리액션/댓글 시)
10. 댓글 → AI 프롬프트 전달
11. 실시간 업데이트 (ActionCable)
12. 정렬 (인기순/최신순)

## 개발 참고사항

### 데이터 모델
- **Project**: organization_id, user_id, opencode_username, is_public, title, description, usage, thumbnail_url
- **Reaction**: project_id, user_id, kind(enum: like/fire/clap/idea), unique index [project_id, user_id, kind]
- **Comment** (2차): project_id, user_id, body, is_read

### 앱 미리보기 URL
- `https://{opencode_username}.{SERVE_DOMAIN}/`
- opencode_username은 users_opencode_ids 테이블에서 매핑
- iframe에 로드, CORS 이미 허용됨 (CHP에서 `Access-Control-Allow-Origin: *`)

### AI 설명 생성
- enk-opencode API로 프로젝트 파일 조회 → LLM에 전달 → 목적/사용법 생성
- 또는 enk-opencode 내 세션 API 활용

### 기존 패턴 참고
- API: JSONAPI + authorize_user_token! (organizations_controller 참고)
- 실시간: OrganizationChannel 패턴 참고
- UI: CDS 컴포넌트 + Emotion + CreateDialog/HackathonProgress 패턴 참고
